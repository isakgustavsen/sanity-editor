The module supports **extending Portable Text** by mapping between:

- TipTap / ProseMirror JSON nodes
- Your custom Portable Text blocks (`PortableTextBlock[]`)

This is done via **`portableTextBlockExtensions`**—an array of objects implementing the
`PortableTextBlockExtension` type.

## What extension modes exist?

There are two supported modes:

- **`mode: 'single'`**: One Portable Text block (`_type`) maps to one (or more) TipTap/ProseMirror node(s).
- **`mode: 'runContainer'`**: A container node (e.g. `taskList`) maps to a *run* of blocks (e.g. consecutive `task` blocks),
  and back again.

See the TypeScript types in [`src/runtime/types/portableTextBlockExtension.ts`](../../../../src/runtime/types/portableTextBlockExtension.ts).

## Example: `task` blocks (run container)

The repository’s playground includes an example extension for a custom Portable Text block type called `task`.

Key idea: the editor stores the UI as a TipTap `taskList` / `taskItem` structure, and converts that into an ordered run of
Portable Text `{ _type: 'task', checked, text }` blocks.

Use the following shape:

```ts
import type { PortableTextBlockExtension } from 'sanity-editor/runtime/types/portableTextBlockExtension'

export const taskBlockExtension: PortableTextBlockExtension = {
  type: 'task',
  mode: 'runContainer',
  containerTiptapNodeType: 'taskList',
  itemTiptapNodeType: 'taskItem',
  schemaDefinition: {
    blockObjects: [
      {
        name: 'task',
        fields: [
          { name: 'checked', type: 'boolean' },
          { name: 'text', type: 'string' },
        ],
      },
    ],
  },

  fromTiptapNode: (node, ctx) => {
    // Convert taskList -> array of Portable Text blocks
  },

  toTiptapNodeRun: (blocks, startIndex, ctx) => {
    // Convert a run of consecutive task blocks -> taskList
  },
}
```

> Note: In this repo, `taskBlockExtension` is intentionally kept in the `playground/` so it is not shipped as part of the module.

## Using extensions in `<PortableTextEditor />`

The module accepts an additional prop:

- `portableTextBlockExtensions`: array of extensions

Example:

```vue
<template>
  <PortableTextEditor
    v-model="blocks"
    :portable-text-block-extensions="[taskBlockExtension]"
  />
</template>

<script setup lang="ts">
import type { PortableTextBlock } from '@portabletext/types'
import { taskBlockExtension } from './taskBlockExtension'

const blocks = ref<PortableTextBlock[]>([])
</script>
```

## How schema compilation works

When you pass `portableTextBlockExtensions`, the module will:

1. Start from the default Portable Text schema definition.
2. Merge your extension schema fragments (e.g. `blockObjects`).
3. Compile the result into a `Schema` used by the conversion layer.

If you do not provide extensions, the module uses the default compiled schema.
