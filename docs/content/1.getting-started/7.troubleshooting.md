## PortableTextEditor doesn’t update your `v-model`

The module intentionally syncs by comparing serialized Portable Text blocks:

- `usePortableTextEditor()` listens for editor updates and emits only when the serialized value changes
- updates from your `v-model` are pushed into TipTap via `editor.commands.setContent(...)`

If you mutate blocks in-place without changing references, Vue reactivity may not trigger the watcher (depending on how you manage state). Prefer replacing the array/object (or ensure you mutate through a reactive ref that triggers updates).

## “Task list” / custom block extensions don’t show up

To render custom Portable Text `_type` blocks, you need to provide:

1. A `PortableTextBlockExtension` in `:portable-text-block-extensions`
2. TipTap extensions that match your mapped node types

The conversion bridge will only map between PT and TipTap node types it knows about.

## Link marks / `link` annotation issues

`sanity-editor` supports a `link` annotation as defined by the default schema:

- Portable Text mark definition: `_type: 'link'` with field `href`
- TipTap node type: `link` mark (`@tiptap/extension-link`)

If your schema doesn’t include the `link` annotation (or you use a different mark definition), conversions will not round-trip correctly unless you also extend the schema/extension mapping.
