:::note
`sanity-editor` is marked **experimental**. This page records practical notes when evolving your Portable Text schema and extensions.
:::

## Upgrading between releases

- Check your installed `nuxt-tiptap-editor` version and ensure TipTap editor extensions still match the node types you rely on (`taskList`, `taskItem`, etc.).
- If you have custom block extensions, re-test round-tripping:
  - `portableTextToTipTapJson()` then `prosemirrorJsonToPortableText()`
  - Confirm you still produce the same `_type` and required fields.

## Changing your Portable Text schema

If you change the schema (decorators/annotations/styles or add/remove block object types), update your editor options accordingly:

1. Provide `schema` (or custom block extensions with `schemaDefinition`) to `usePortableTextEditor`.
2. Ensure your `PortableTextBlockExtension` objects:
   - Use the correct `_type` (Portable Text)
   - Use the correct TipTap node types (`tiptapNodeTypes` or container/item types)

## Token keys (`_key`) differences

`sanity-editor` uses a key generator for Portable Text blocks/spans created during conversion.
If your app depends on stable `_key` values (e.g. for diffing), pass a custom `keyGenerator`.
