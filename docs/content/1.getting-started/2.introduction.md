`sanity-editor` is an experimental [Nuxt 4](https://nuxt.com) module for editing [Portable Text](https://portabletext.org/) in the browser with [TipTap](https://tiptap.dev/). It keeps your model value as `PortableTextBlock[]` instead of HTML.

## What is Portable Text?

Portable Text is a JSON-based rich text format used heavily in the [Sanity](https://www.sanity.io/) ecosystem: blocks, spans, marks, and custom block types. It is **not HTML**—it is structured data you can validate, query, and render on any platform.

## Why not HTML?

Many WYSIWYG editors round-trip through HTML. That works for generic pages, but Portable Text workflows usually need a stable block model aligned with `@portabletext/schema`. This module bridges TipTap/ProseMirror JSON and Portable Text so your `v-model` stays `PortableTextBlock[]` end to end.

## Experimental status

The API, schema coverage, and real-world Sanity workflows are still evolving. Please share feedback via [GitHub Issues](https://github.com/isakgustavsen/sanity-editor/issues) or [Discussions](https://github.com/isakgustavsen/sanity-editor/discussions).

## Requirements

- **Nuxt** `^4`
- **`sanity-editor`** — **`nuxt-tiptap-editor`** is declared as a module dependency and registered for you (see [Installation](/getting-started/installation))
