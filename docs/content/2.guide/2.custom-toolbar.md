## `#toolbar` slot

`SanityEditor` renders a toolbar row above the content. By default it shows simple **Bold**, **Italic**, **H2**, and **List** buttons.

Override the slot to build your own UI. The slot receives **`{ editor }`** — a TipTap **`Editor`** instance (may be `null` before mount; the template only shows the toolbar when `editor` exists).

```vue [MyEditor.vue]
<template>
  <SanityEditor v-model="content">
    <template #toolbar="{ editor }">
      <button
        type="button"
        @click="editor.chain().focus().toggleBold().run()"
      >
        Bold
      </button>
      <button
        type="button"
        @click="editor.chain().focus().toggleItalic().run()"
      >
        Italic
      </button>
      <button
        type="button"
        @click="editor.chain().focus().toggleHeading({ level: 3 }).run()"
      >
        H3
      </button>
      <button
        type="button"
        @click="editor.chain().focus().toggleBulletList().run()"
      >
        Bullet list
      </button>
      <button
        type="button"
        @click="editor.chain().focus().setHorizontalRule().run()"
      >
        Horizontal rule
      </button>
    </template>
  </SanityEditor>
</template>
```

Use any TipTap commands your extensions support—`editor.chain().focus()...` is the usual pattern.

## `useSanityEditor` for fully custom chrome

If you do not use `SanityEditor` at all, call **`useSanityEditor`** with your own template and render TipTap content with **`TiptapEditorContent`** from `nuxt-tiptap-editor` (see the playground and [Composable](/api/composable)).
