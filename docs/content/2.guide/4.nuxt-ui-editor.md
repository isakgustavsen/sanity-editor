Use **`UEditorToolbar`** and other child components from [Nuxt UI's editor](https://ui.nuxt.com/docs/components/editor) with **`usePortableTextEditor`** to build a Portable Text editor with Nuxt UI styling. Those components accept a standard TipTap **`Editor`** as a prop — which is exactly what the composable returns.

::warning
Do not use **`UEditor`** (the root component) as a Portable Text editor. It creates its own TipTap instance and models **HTML, Markdown, or generic JSON** — not `PortableTextBlock[]`. Only the child components (`UEditorToolbar`, `UEditorSuggestionMenu`, `UEditorDragHandle`, etc.) are compatible with `usePortableTextEditor`.
::

## Set up prerequisites

- **`@nuxt/ui`** installed and registered in your Nuxt app — see [Nuxt UI installation](https://ui.nuxt.com/docs/getting-started/installation).
- **`sanity-editor`** and **`nuxt-tiptap-editor`** installed, with `sanity-editor` added to `modules` — see [Installation](/getting-started/installation).

## Build the editor layout

Call `usePortableTextEditor`, then pass the returned `editor` ref to both `UEditorToolbar` and `TiptapEditorContent`:

```vue [MyEditor.vue]
<script setup lang="ts">
import type { PortableTextBlock } from '@portabletext/types'
import type { EditorToolbarItem } from '@nuxt/ui'

const content = ref<PortableTextBlock[]>([])

const { editor } = usePortableTextEditor(content, (value) => {
  content.value = value
})

const toolbarItems: EditorToolbarItem[][] = [[
  { kind: 'mark', mark: 'bold', icon: 'i-lucide-bold', tooltip: { text: 'Bold' } },
  { kind: 'mark', mark: 'italic', icon: 'i-lucide-italic', tooltip: { text: 'Italic' } },
  { kind: 'heading', level: 2, icon: 'i-lucide-heading-2', tooltip: { text: 'Heading 2' } },
  { kind: 'bulletList', icon: 'i-lucide-list', tooltip: { text: 'Bullet list' } },
  { kind: 'horizontalRule', icon: 'i-lucide-separator-horizontal', tooltip: { text: 'Horizontal rule' } },
]]
</script>

<template>
  <div class="rounded-lg border border-default overflow-hidden">
    <UEditorToolbar
      v-if="editor"
      :editor="editor"
      :items="toolbarItems"
      class="border-b border-muted px-2 py-2"
    />
    <TiptapEditorContent
      v-if="editor"
      :editor="editor"
      class="p-4 min-h-[12rem]"
    />
  </div>
</template>
```

::note
Toolbar `kind` values must match TipTap commands on your editor instance. The module includes **StarterKit** (headings 1–6) and **Link** by default. Pass extra `extensions` to `usePortableTextEditor` for any Nuxt UI items that require additional nodes — for example underline, task lists, or text align.
::

## Add link popover and suggestion menus

`UEditorToolbar` exposes a **`#link`** slot for a custom link popover. Use the same `editor` ref from `usePortableTextEditor` to wire it. The same approach works for **`UEditorSuggestionMenu`** (slash commands), **`UEditorMentionMenu`** (@-mentions), bubble toolbars, and `UEditorDragHandle` — all take an `editor` prop. See the [Nuxt UI Editor examples](https://ui.nuxt.com/docs/components/editor) for full usage.

## Resolve ProseMirror plugin conflicts

If you see **"Adding different instances of a keyed plugin"** after combining `@nuxt/ui` with TipTap, pre-bundle the ProseMirror packages in your Nuxt config:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  vite: {
    optimizeDeps: {
      include: [
        '@nuxt/ui > prosemirror-state',
        '@nuxt/ui > prosemirror-transform',
        '@nuxt/ui > prosemirror-model',
        '@nuxt/ui > prosemirror-view',
        '@nuxt/ui > prosemirror-gapcursor',
      ],
    },
  },
})
```

## Choose the right approach

| Approach | Use when |
| --- | --- |
| `PortableTextEditor` | You want the built-in toolbar and minimal setup. |
| `usePortableTextEditor` + `TiptapEditorContent` + custom HTML | Full control without Nuxt UI. |
| `usePortableTextEditor` + `TiptapEditorContent` + `UEditorToolbar` | Portable Text data model with Nuxt UI's editor UX. |
| `UEditor` alone | You do **not** need Portable Text — use HTML, Markdown, or JSON as in the Nuxt UI docs. |
