Use Nuxt UIŌĆÖs **[`UEditor`](https://ui.nuxt.com/docs/components/editor)** with **`content-type="json"`** and the moduleŌĆÖs conversion helpers. Call **`createSanityEditorContext()`** once, then **`sanityEditorBlocksToTiptapJson`** for the bound value and **`sanityEditorProsemirrorJsonToBlocks`** on **`@update:model-value`**. You own the two-way boundary; the module does not wrap `UEditor`.

## Set up prerequisites

- **`@nuxt/ui`** installed and registered ŌĆö see [Nuxt UI installation](https://ui.nuxt.com/docs/getting-started/installation).
- **`sanity-editor`** and **`nuxt-tiptap-editor`** with `sanity-editor` in `modules` ŌĆö see [Installation](/getting-started/installation).

## Bind `UEditor` to Portable Text

```vue [MyEditor.vue]
<script setup lang="ts">
import type { PortableTextBlock } from '@portabletext/types'
import type { JSONContent } from '@tiptap/vue-3'

const content = ref<PortableTextBlock[]>([])
const ctx = createSanityEditorContext()

const tiptapJson = computed(() => sanityEditorBlocksToTiptapJson(content.value, ctx))

function handleUpdate(value: JSONContent) {
  content.value = sanityEditorProsemirrorJsonToBlocks(value, ctx)
}
</script>

<template>
  <UEditor
    :model-value="tiptapJson"
    content-type="json"
    @update:model-value="handleUpdate"
  />
</template>
```

`content` stays **`PortableTextBlock[]`**. When it changes from the outside (e.g. after a fetch), the computed doc updates and `UEditor` receives new JSON.

::note
**`UEditor`** ships extra marks and nodes (e.g. underline) that may not exist in the default [`@portabletext/schema`](https://www.npmjs.com/package/@portabletext/schema) definition. Extend the schema and conversion layer if you enable features beyond what [Using the editor](/guide/using-the-editor) documents, or restrict toolbar / StarterKit so TipTap output stays mappable.
::

## Resolve ProseMirror plugin conflicts

If you see **"Adding different instances of a keyed plugin"** after combining `@nuxt/ui` with TipTap, pre-bundle ProseMirror in Nuxt:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  vite: {
    optimizeDeps: {
      include: [
        '@nuxt/ui > prosemirror-state',
        '@nuxt/ui > prosemirror-transform',
        '@nuxt/ui > prosemirror-model',
        '@nuxt/ui > prosemirror-view',
        '@nuxt/ui > prosemirror-gapcursor',
      ],
    },
  },
})
```

## Alternative: `useSanityEditor` + toolbar chrome

If you prefer a **single TipTap instance** shared with **`TiptapEditorContent`** and Nuxt UIŌĆÖs **`UEditorToolbar`** (or other components that take an `editor` ref), use **`useSanityEditor`** and pass the returned **`editor`** ŌĆö see [Custom toolbar](/guide/custom-toolbar).

## Choose the right approach

| Approach | Use when |
| --- | --- |
| `SanityEditor` | Built-in toolbar and minimal setup. |
| `UEditor` + `createSanityEditorContext` + conversion functions | Full Nuxt UI editor (toolbars, slash menu, etc.) with `PortableTextBlock[]` at the boundary. |
| `useSanityEditor` + `TiptapEditorContent` + `UEditorToolbar` | Same composable as `SanityEditor`, custom layout with Nuxt UI chrome. |
| `useSanityEditor` + `TiptapEditorContent` + plain HTML | No Nuxt UI. |
| `UEditor` without conversions | You do **not** need Portable Text ŌĆö use HTML, Markdown, or JSON as in the Nuxt UI docs. |
