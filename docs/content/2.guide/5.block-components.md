## Portable Text, not MDC

**sanity-editor** stores **`PortableTextBlock[]`**. Tools that serialize to **MDC** or Markdown use a different pipeline; here, a “custom component” means:

1. A TipTap **block-level** node (your own **`Node.create({ ... })`** or a community extension).
2. A **`SanityEditorBlockExtension`** so TipTap JSON ↔ Portable Text stays in sync.

For **list-like containers** (one TipTap parent, many items → many PT blocks), use **`mode: 'runContainer'`** by hand — see [Block extensions](/guide/block-extensions).

## `defineSanityEditorBlockComponent`

Auto-imported helper that builds a **`single`** extension from:

| Option | Purpose |
| --- | --- |
| **`type`** | Portable Text block object **`_type`** |
| **`tiptapNodeType`** | One or more TipTap root node names (string or array) |
| **`schemaDefinition`** | Optional fragment merged into the default schema (e.g. **`blockObjects`**) |
| **`fromTiptapNode`** | TipTap JSON node → **`PortableTextBlock[] \| null`** |
| **`toTiptapNode`** | One **`PortableTextBlock`** → TipTap JSON **`null`** if not yours |

Pair it with your TipTap **`extensions`** and pass the result in **`blockExtensions`** on **`SanityEditor`**, **`useSanityEditor`**, or **`createSanityEditorContext`**.

## Conversion helpers

- **`sanityEditorGenerateKey(ctx)`** — next **`_key`** using the context’s **`keyGenerator`** (or the default).
- **`sanityEditorFlattenTextInTiptapNode(node)`** — plain text from a TipTap JSON subtree (paragraphs, **`hardBreak`** → newlines). Useful when the PT shape uses simple string fields.

## Schema resolution

**`useSanityEditor`** and **`createSanityEditorContext`** both use **`resolveSanityEditorCompiledSchema`**: an explicit **`schema`** wins; otherwise, when **`blockExtensions`** is non-empty, the default definition is **merged** with each extension’s **`schemaDefinition`** (deduped by name) and compiled. That keeps validation-oriented schema aligned with custom block objects.

To merge definitions yourself (e.g. tests or custom tooling), use **`mergeSchemaDefinitions`**.

## `SanityEditor` and `UEditor`

Use the **same** **`blockExtensions`** and matching TipTap **`extensions`** everywhere you edit the same document:

- **`SanityEditor`** / **`useSanityEditor`** — pass **`blockExtensions`** and **`extensions`** as props / options.
- **Nuxt UI `UEditor`** — **`createSanityEditorContext({ blockExtensions, keyGenerator })`**, bind **`:model-value="sanityEditorBlocksToTiptapJson(blocks, ctx)"`**, and in **`@update:model-value`** assign **`sanityEditorProsemirrorJsonToBlocks(value, ctx)`**. Add your node extension to **`UEditor`**’s **`extensions`** array.

## Reference: callout (playground)

**`playground/calloutExtension.ts`** defines a **`callout`** TipTap block and a matching PT block object (`variant`, `text`). **`playground/pages/index.vue`** uses **`SanityEditor`**; **`playground/pages/u-editor.vue`** shows **`UEditor`** with **`model-value`** sync and an insert button.
