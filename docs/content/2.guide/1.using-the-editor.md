## `PortableTextEditor` props

| Prop | Type | Description |
| --- | --- | --- |
| `v-model` | `PortableTextBlock[]` | Two-way binding for Portable Text blocks. |
| `schema` | `Schema` (optional) | Compiled `@portabletext/schema` schema. Defaults to the built-in compiled default schema, or is merged when `portableTextBlockExtensions` are provided. |
| `extensions` | `AnyExtension[]` (optional) | Extra TipTap extensions appended after the base set (e.g. task list). |
| `portableTextBlockExtensions` | `PortableTextBlockExtension[]` (optional) | Custom mappings between TipTap nodes and Portable Text block types. |
| `editorProps` | `Record<string, unknown>` (optional) | Extra props passed into TipTap’s `useEditor`. Do not set `immediatelyRender` or `onUpdate` (the composable owns those). |
| `keyGenerator` | `() => string` (optional) | Generates `_key` values for new blocks during conversion. |

See also: [API Reference — PortableTextEditor](/api/portable-text-editor).

## Default TipTap extensions

The composable always applies:

- **`StarterKit`** — headings **levels 1–6**, paragraphs, lists, blockquote, code block, horizontal rule, etc.
- **`Link`** — `openOnClick: false`, `rel: "noopener noreferrer"` on links.

Your **`extensions`** are merged **after** these.

## Default Portable Text schema

The default definition matches common blog-style body content and aligns with the transformers in `prosemirror-portable-text`:

- **Styles:** `normal`, `h1`–`h6`, `blockquote`
- **Lists:** `bullet`, `number`
- **Decorators:** `strong`, `em`, `strike-through`, `code`
- **Annotations:** `link` (field: `href`, type `string`)
- **Block objects:** `horizontal-rule`
- **Inline objects:** none

Exports: `defaultPortableTextSchemaDefinition` and `defaultCompiledPortableTextSchema`.
