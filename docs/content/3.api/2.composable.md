## Overview

Use `useSanityEditor` when you want the same Portable Text sync behavior as `SanityEditor` but with your own layout and toolbar UI.

## Signature

```ts
function useSanityEditor(
  modelValue: Ref<PortableTextBlock[]>,
  emitUpdate: (value: PortableTextBlock[]) => void,
  options?: UseSanityEditorOptions,
): { editor: Ref<Editor | undefined> }
```

Pass a ref to your model value and an update callback. The composable handles two-way conversion between TipTap JSON and `PortableTextBlock[]` for both editor updates and external model changes.

## Basic usage

```vue [MyEditor.vue]
<script setup lang="ts">
import type { PortableTextBlock } from '@portabletext/types'

const content = ref<PortableTextBlock[]>([])
const { editor } = useSanityEditor(content, (value) => {
  content.value = value
})
</script>

<template>
  <TiptapEditorContent v-if="editor" :editor="editor" />
</template>
```

## `UseSanityEditorOptions`

| Option | Type | Description |
| --- | --- | --- |
| `schema` | `Schema` | Defaults to `sanityEditorDefaultCompiledSchema`, or merged with `blockExtensions` when those are set. |
| `extensions` | `AnyExtension[]` | Appended after StarterKit + Link. |
| `blockExtensions` | `SanityEditorBlockExtension[]` | Custom block mappings. |
| `editorProps` | `Record<string, unknown>` | Spread into `useEditor` **after** internal options. **Do not** set `immediatelyRender` or `onUpdate`—the composable sets `immediatelyRender: false` and owns `onUpdate` for sync. |
| `keyGenerator` | `() => string` | Passed into the transform context for `_key` generation. |

## Return value

- **`editor`** — Vue ref to the TipTap **`Editor`** instance (destroyed on unmount).

## Notes

- The composable appends your `extensions` after `StarterKit` and `Link`.
- Schema behavior matches `createSanityEditorContext`: explicit `schema` wins; otherwise block extension schema fragments are merged and compiled.
