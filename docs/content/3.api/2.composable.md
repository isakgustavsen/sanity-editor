## Signature

```ts
function usePortableTextEditor(
  modelValue: Ref<PortableTextBlock[]>,
  emitUpdate: (value: PortableTextBlock[]) => void,
  options?: UsePortableTextEditorOptions,
): { editor: Ref<Editor | undefined> }
```

Pass a **ref** to your `v-model` array and a setter that updates the parent (e.g. from `defineEmits` or `model.value = …`). The composable keeps TipTap JSON and Portable Text in sync on update and when `modelValue` changes (deep watch).

## `UsePortableTextEditorOptions`

| Option | Type | Description |
| --- | --- | --- |
| `schema` | `Schema` | Defaults to `defaultCompiledPortableTextSchema`, or merged with `portableTextBlockExtensions` when those are set. |
| `extensions` | `AnyExtension[]` | Appended after StarterKit + Link. |
| `portableTextBlockExtensions` | `PortableTextBlockExtension[]` | Custom block mappings. |
| `editorProps` | `Record<string, unknown>` | Spread into `useEditor` **after** internal options. **Do not** set `immediatelyRender` or `onUpdate`—the composable sets `immediatelyRender: false` and owns `onUpdate` for PT sync. |
| `keyGenerator` | `() => string` | Passed into the transform context for `_key` generation. |

## Return value

- **`editor`** — Vue ref to the TipTap **`Editor`** instance (destroyed on unmount).

## Usage in your own component

Mirror the pattern in `PortableTextEditor.vue`: call the composable, render **`TiptapEditorContent`** from `nuxt-tiptap-editor` with `:editor="editor"`, and implement your own toolbar using `editor.value?.chain()...`.
