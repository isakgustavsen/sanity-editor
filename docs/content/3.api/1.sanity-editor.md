## Overview

`SanityEditor` is the fastest way to edit `PortableTextBlock[]` in a Nuxt app. It wires `useSanityEditor` to `TiptapEditorContent` and ships with a simple default toolbar.

## Basic usage

```vue [MyEditor.vue]
<script setup lang="ts">
import type { PortableTextBlock } from '@portabletext/types'

const content = ref<PortableTextBlock[]>([])
</script>

<template>
  <SanityEditor v-model="content" />
</template>
```

## Slots

| Slot | Scope | Description |
| --- | --- | --- |
| `toolbar` | `{ editor: Editor \| null }` | Replaces the default Bold / Italic / H2 / List toolbar. Only rendered when `editor` is truthy. |

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `v-model` | `PortableTextBlock[]` | `[]` | Blocks to edit (Portable Text shape from `@portabletext/types`). |
| `schema` | `Schema` | — | Compiled `@portabletext/schema` schema. |
| `extensions` | `AnyExtension[]` | — | Additional TipTap extensions. |
| `blockExtensions` | `SanityEditorBlockExtension[]` | — | Custom TipTap ↔ block mappings. |
| `editorProps` | `Record<string, unknown>` | — | Extra TipTap editor options (see composable). |
| `keyGenerator` | `() => string` | — | Key generator for new `_key` values in conversion. |

## Notes

- Use `blockExtensions` and matching TipTap `extensions` together when adding custom block components.
- Prefer `keyGenerator` when you need deterministic `_key` generation in tests or controlled migration flows.
