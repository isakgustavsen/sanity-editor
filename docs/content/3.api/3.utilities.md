## Overview

Use utilities when you are not relying only on `SanityEditor`, or when you need explicit control over conversion and schema behavior.

## Choose the right helper

| Need | Helper |
| --- | --- |
| Build transform context for custom editors | `createSanityEditorContext` |
| Convert TipTap JSON to `PortableTextBlock[]` | `sanityEditorProsemirrorJsonToBlocks` |
| Convert `PortableTextBlock[]` to TipTap JSON | `sanityEditorBlocksToTiptapJson` |
| Define a single-node block component mapping | `defineSanityEditorBlockComponent` |
| Resolve compiled schema from options/extensions | `resolveSanityEditorCompiledSchema` |

## `createSanityEditorContext`

```ts
function createSanityEditorContext(
  options?: CreateSanityEditorContextOptions,
): SanityEditorTransformContext

interface CreateSanityEditorContextOptions {
  schema?: Schema
  keyGenerator?: () => string
  blockExtensions?: SanityEditorBlockExtension[]
}
```

Returns a **`SanityEditorTransformContext`** for **`sanityEditorBlocksToTiptapJson`** and **`sanityEditorProsemirrorJsonToBlocks`**.

**Schema behavior** matches **`useSanityEditor`**: if **`schema`** is omitted, the default compiled schema is used, **unless** **`blockExtensions`** is non-empty — then the default **`defineSchema`** output is **merged** with each extension’s **`schemaDefinition`** and **compiled** (see **`resolveSanityEditorCompiledSchema`**).

Use this when you integrate your own editor surface (for example Nuxt UI `UEditor`) instead of `useSanityEditor`.

## Schema resolution

### `resolveSanityEditorCompiledSchema`

```ts
function resolveSanityEditorCompiledSchema(
  options?: ResolveSanityEditorCompiledSchemaOptions,
): Schema

interface ResolveSanityEditorCompiledSchemaOptions {
  schema?: Schema
  blockExtensions?: SanityEditorBlockExtension[]
}
```

- If **`options.schema`** is set, it is returned unchanged.
- Otherwise, if **`blockExtensions`** has items, merges **`sanityEditorDefaultSchemaDefinition`** with each extension’s **`schemaDefinition`** and **`compileSchema`** the result.
- Otherwise returns **`sanityEditorDefaultCompiledSchema`**.

### `mergeSchemaDefinitions`

```ts
function mergeSchemaDefinitions(
  base: SchemaDefinition,
  additions: Array<SchemaDefinition | undefined>,
): SchemaDefinition
```

Merges Portable Text schema fragments (styles, lists, decorators, annotations, block objects, inline objects), **deduping by `name`**. Useful when composing schema definitions outside the editor.

## Block component helper

### `defineSanityEditorBlockComponent`

```ts
function defineSanityEditorBlockComponent(
  options: DefineSanityEditorBlockComponentOptions,
): SanityEditorSingleNodeBlockExtension
```

Builds a `single` `SanityEditorBlockExtension` from TipTap node type name(s) and `fromTiptapNode` / `toTiptapNode` callbacks. See [Custom block components](/guide/block-components).

## Conversion helpers

### `sanityEditorProsemirrorJsonToBlocks`

```ts
function sanityEditorProsemirrorJsonToBlocks(
  docJson: unknown,
  ctx: SanityEditorTransformContext,
): PortableTextBlock[]
```

Converts a TipTap **`editor.getJSON()`** document (root **`type: 'doc'`**) into an array of **`PortableTextBlock`**.

### `sanityEditorBlocksToTiptapJson`

```ts
function sanityEditorBlocksToTiptapJson(
  blocks: PortableTextBlock[],
  ctx: SanityEditorTransformContext,
): TiptapJSONDoc
```

Converts blocks into TipTap/ProseMirror JSON suitable for **`setContent`**.

### `sanityEditorGenerateKey`

```ts
function sanityEditorGenerateKey(ctx: SanityEditorTransformContext): string
```

Returns the next Portable Text **`_key`** using **`ctx.keyGenerator`** or the built-in default.

### `sanityEditorFlattenTextInTiptapNode`

```ts
function sanityEditorFlattenTextInTiptapNode(node: TiptapJSONNode): string
```

Plain text from a TipTap JSON node tree (e.g. paragraph children, **`hardBreak`** as newlines). Handy inside **`fromTiptapNode`** when your block object stores a simple string field.

## `SanityEditorTransformContext`

| Field | Type | Description |
| --- | --- | --- |
| `schema` | `Schema` | Compiled schema (from **`resolveSanityEditorCompiledSchema`** or manual). |
| `keyGenerator` | `() => string` | Optional; defaults to random base36 keys. |
| `blockExtensions` | `SanityEditorBlockExtension[]` | Optional custom block mappings. |

## JSON types

Exported for typing:

- **`TiptapJSONDoc`** — `{ type: 'doc', content?: TiptapJSONNode[] }`
- **`TiptapJSONNode`**, **`TiptapJSONMark`**

## Default schema exports

| Export | Description |
| --- | --- |
| `sanityEditorDefaultSchemaDefinition` | `defineSchema({ ... })` from `@portabletext/schema`. |
| `sanityEditorDefaultCompiledSchema` | `compileSchema(sanityEditorDefaultSchemaDefinition)` — use as default **`schema`** in context. |

Use these when you call conversion helpers outside the editor. Prefer `createSanityEditorContext()` over assembling context by hand when `blockExtensions` should drive schema merging.
