## `createSanityEditorContext`

```ts
function createSanityEditorContext(
  options?: CreateSanityEditorContextOptions,
): SanityEditorTransformContext

interface CreateSanityEditorContextOptions {
  schema?: Schema
  keyGenerator?: () => string
  blockExtensions?: SanityEditorBlockExtension[]
}
```

Returns a **`SanityEditorTransformContext`** for **`sanityEditorBlocksToTiptapJson`** and **`sanityEditorProsemirrorJsonToBlocks`**, defaulting **`schema`** to **`sanityEditorDefaultCompiledSchema`**. Use this when you integrate your own editor surface (for example Nuxt UI **`UEditor`**) instead of **`useSanityEditor`**.

## Conversion functions

### `sanityEditorProsemirrorJsonToBlocks`

```ts
function sanityEditorProsemirrorJsonToBlocks(
  docJson: unknown,
  ctx: SanityEditorTransformContext,
): PortableTextBlock[]
```

Converts a TipTap **`editor.getJSON()`** document (root `type: 'doc'`) into an array of **`PortableTextBlock`**.

### `sanityEditorBlocksToTiptapJson`

```ts
function sanityEditorBlocksToTiptapJson(
  blocks: PortableTextBlock[],
  ctx: SanityEditorTransformContext,
): TiptapJSONDoc
```

Converts blocks into TipTap/ProseMirror JSON suitable for **`setContent`**.

### `SanityEditorTransformContext`

| Field | Type | Description |
| --- | --- | --- |
| `schema` | `Schema` | Required compiled schema for validation-style transforms. |
| `keyGenerator` | `() => string` | Optional; defaults to random base36 keys. |
| `blockExtensions` | `SanityEditorBlockExtension[]` | Optional custom block mappings. |

## JSON types

Exported for typing:

- **`TiptapJSONDoc`** — `{ type: 'doc', content?: TiptapJSONNode[] }`
- **`TiptapJSONNode`**, **`TiptapJSONMark`**

## Default schema exports

| Export | Description |
| --- | --- |
| `sanityEditorDefaultSchemaDefinition` | `defineSchema({ ... })` from `@portabletext/schema`. |
| `sanityEditorDefaultCompiledSchema` | `compileSchema(sanityEditorDefaultSchemaDefinition)` — use as default **`schema`** in context. |

Use these when you call the conversion helpers outside the editor. Prefer **`createSanityEditorContext()`** over assembling the context object by hand.
